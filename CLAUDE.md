# CLAUDE.md — планировщик стабильности (форк TeymiaHabit)

## Что это за проект
Персональный iOS-планировщик, спроектированный под худший день, чтобы работать каждый день. Форк TeymiaHabit (MIT, SwiftUI + SwiftData + WidgetKit, iOS 18+). Единственный пользователь фазы 1 — Олег.

## Обязательное чтение перед КАЖДОЙ сессией
1. `docs/planner-spec-v1.md` — спецификация (источник правды по фичам)
2. `docs/productivity-methodologies-knowledge-base.md` — база знаний (принципы П1–П20, антирекомендации А1–А11)
3. `docs/PROJECT-MAP.md` — карта кодовой базы (обновляется в конце каждой сессии)

## Несущие принципы (нарушать нельзя)
- Проектируем для пользователя с 5% батарейки. Каждое решение проверяется вопросом: «Сработает ли в день, когда всё плохо?»
- Уведомление требует решения (Сделано / Мини / Перенести / Отменить), не подвига.
- Стрики, красная просрочка, счётчики долга, накопительная геймификация с потерями — ЗАПРЕЩЕНЫ (А1, А3, А11).
- doneMini визуально идентичен doneFull. Никаких серых галочек и процентов выполнения задачи.
- В UI не существуют слова: провал, сорвался, просрочено, ленился, опять.
- Ритуалы ≤5 минут. Ни один пропуск ритуала не создаёт долга.
- По умолчанию приложение решает само, пользователь может переопределить. Никогда наоборот.
- Классы напоминаний: .routine (1 пуш) / .focus (≤2 пуша-выбора) / .critical (эскалация 15 мин, максимум 2–3 в сутки, валидация в UI).
- Лимит iOS: 64 pending-уведомления → перепланирование очереди при foreground + BGAppRefreshTask.

## Правила работы в сессиях
- Одна сессия = один модуль из плана (раздел 12 спеки). Не перепрыгивать.
- Минимальные диффы. Не рефакторить несвязанное. Модуль привычек и уведомлений TeymiaHabit — переиспользовать, не переписывать.
- После каждого значимого изменения — сборка (xcodebuild / симулятор). Не накапливать несобранное.
- В конце сессии: обновить `docs/PROJECT-MAP.md` (что где лежит, что изменено) + короткое резюме для следующей сессии.
- Коммиты: маленькие, по-русски, формат «сессия N: что сделано».
- Если требование сессии конфликтует со спекой или базой знаний — остановиться и спросить, не решать самовольно.

## Технический стек
SwiftUI, SwiftData, UserNotifications (+ notification actions), WidgetKit, iOS 18+. Хранение локальное, офлайн-first. CloudKit — не подключать (фаза 2). RevenueCat — выпилен (сессия 1). Язык UI — русский, хардкод.

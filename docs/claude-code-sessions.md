# Промты сессий Claude Code — копировать по одному, строго по порядку

## Перед сессией 1 — руками, 15 минут (Claude Code не нужен)
1. Форкнуть https://github.com/amanbayserkeev0377/TeymiaHabit → `git clone` на Mac.
2. Создать в репо папку `docs/`, положить туда: `planner-spec-v1.md`, `productivity-methodologies-knowledge-base.md`. `CLAUDE.md` — в корень репо.
3. Открыть в Xcode, выбрать свой Apple ID (free provisioning), собрать на iPhone КАК ЕСТЬ, ничего не меняя. Приложение запустилось на телефоне = чекпоинт пройден.
Если сборка падает — не чинить руками, это первая задача сессии 1.

---

## Сессия 1 — чистка (1 вечер)
```
Прочитай CLAUDE.md и оба документа в docs/. Это сессия 1.

1. Изучи структуру проекта и создай docs/PROJECT-MAP.md: где модели, view, notification-логика, где RevenueCat/пейволл, точки входа.
2. [Если сборка на устройстве падала — сначала почини сборку, опиши что было.]
3. Полностью выпили RevenueCat и логику подписок: зависимость из SPM, пейволл-экраны, проверки Pro-статуса. Все Pro-фичи (мультинапоминания и пр.) делаются доступными безусловно.
4. Собери проект, убедись что билд зелёный.
5. Ничего нового не добавляй. В конце: список изменённых файлов + обнови PROJECT-MAP.md + резюме для сессии 2.
```

## Сессия 2 — модель задач и Today (1 вечер)
```
Прочитай CLAUDE.md, docs/PROJECT-MAP.md и раздел 3–4 спеки. Это сессия 2.

1. Создай SwiftData-модель TaskItem по разделу 3 спеки (все поля, включая minimalVersion, lifeArea, status, priority/NagClass, anchorEvent, moveCount, isAnchorOfDay). Enum LifeArea: work/personal/friends/health/recovery. Enum TaskStatus по спеке — статусы нейтральные, слова «просрочено/провал» не использовать нигде, включая имена в коде UI-текстов.
2. Экран Today: секции по сферам, якорная сверху со звездой, добавление задачи (пока простое: текст + сфера + дедлайн опционально + мини-версия).
3. Правило отображения: doneMini и doneFull — одна и та же галочка.
4. Таб-бар: Today + существующие привычки TeymiaHabit. Ничего из механики привычек не трогать.
5. Уведомления для задач пока НЕ делать (сессия 3). Сборка зелёная, обнови PROJECT-MAP.md, резюме.
```

## Сессия 3 — NagEngine (1–2 вечера)
```
Прочитай CLAUDE.md, PROJECT-MAP.md и раздел 4.5 спеки. Это сессия 3.

1. Сервис NagEngine: планирование уведомлений по классам —
   .routine: 1 пуш + badge, без эскалации;
   .focus: до 2 пушей/день, текст-выбор;
   .critical: повтор каждые 15 мин до принятия решения; валидация в UI: не более 3 задач .critical на день.
2. UNNotificationAction на уведомлениях: «Сделано / Мини / +1 час» — обрабатываются без открытия приложения.
3. Очередь: перепланирование при foreground + BGAppRefreshTask, лимит 64 pending.
4. Правило двух пропусков и правило трёх переносов (moveCount >= 3 → диалог «уменьшить/удалить/слот») — по спеке.
5. Тексты уведомлений — по разделу 10 спеки (формула: нормализация → действие → уменьшение). Сборка зелёная, PROJECT-MAP.md, резюме.
```

## После сессии 3
У тебя рабочая v0.1: задачи + сферы + привычки + напоминания трёх классов. C этого момента приложение ставится «в бой» параллельно с доработкой. Промты сессий 4+ (шлюз/амнистия, capture, типы дней…) — запросить у Claude в чате, принеся актуальный PROJECT-MAP.md.

## Правила процесса (напоминание себе)
- Один вечер = одна сессия. Не начинать следующую в тот же вечер, даже если прёт.
- После 22:30 — стоп, это правило системы, которую ты строишь.
- Застрял >30 минут на одной ошибке → скопировать ошибку и PROJECT-MAP.md в чат Claude, не бодаться в одиночку.

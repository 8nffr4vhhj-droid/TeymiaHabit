**Версия 1.0 · июль 2026 · единый источник правды для сессий Claude Code**

## 0. Философия (читать перед каждой сессией)

Приложение проектируется **для пользователя-уставшего с 5% батарейки**, а не для его сильной версии. Каждое проектное решение проверяется вопросом: *«Сработает ли это в день, когда всё плохо?»*

Три несущих принципа:
1. **Система давит на ясность, а не на выполнение.** Уведомление требует решения (сделать / мини / перенести / отменить), а не подвига.
2. **Пропуск — норма. Главный навык — возврат** (П4). Празднуем возврат сильнее регулярности.
3. **По умолчанию приложение решает само, пользователь может переопределить** (П3). Никогда наоборот.

Любой фейл пользователя интерпретируется по модели B=MAP (Фогг): сначала чиним триггер (Prompt), потом упрощаем (Ability), и никогда не объясняем «слабой волей» (Motivation).

## 2. Типы дней и лимиты (П2)

Тип дня выбирается утром одним тапом (или наследуется из расписания недели). Лимиты жёсткие: UI физически не даёт добавить сверх — только через zero-sum свап «что убрать» (П18).

| Тип дня | Лимит задач | Якорная | Поведение системы |
|---|---|---|---|
| Выживание | 1 | — | Показывает только мини-версии + рутины здоровья |
| Плохой | 1–2 | опц. | Только мини-версии, тон максимально бережный |
| Обычный будень | 3 | 1 | Стандарт |
| Сильный | до 5 | 1 | Доступен opt-in «сначала главное» (А4) |
| Подработка (Сб) | 1 блок + 1 бытовая | блок | Рабочий контекст «спит» |
| Воскресенье | ревью + отдых | — | Только ритуалы, задачи не планируются |

- **Якорная задача** — одна, со звездой: «сделал только её — день зачтён». Выполнена якорная → день визуально успешный, сколько бы ни осталось.
- Слот 18:00–19:00 (транзит/шлюз) — задачи на это время не ставятся.
- Категория «восстановление» (сон, прогулка, еда) — задачи первого класса, учитываются в лимите (П7).

## 3. Модель данных (SwiftData)

```
TaskItem
  title: String
  minimalVersion: String?      // обязательна для повторяющихся и страшных задач
  lifeArea: LifeArea           // .work / .personal / .friends / .health / .recovery
  status: TaskStatus           // .inbox / .planned / .started / .doneFull / .doneMini
                               // / .moved / .cancelled / .noResource / .someday
  priority: NagClass           // .routine / .focus / .critical
  deadline: Date?
  anchorEvent: String?         // «после кофе», «после прихода домой» (П6)
  prepNote: String?            // «что подготовить заранее» (П11)
  estimateMinutes: Int?        // при планировании умножается на 1.5 (П20)
  moveCount: Int               // счётчик переносов
  isAnchorOfDay: Bool
  goalID: UUID?                // связь с целью, опциональна

Habit (существующая модель TeymiaHabit, расширяется)
  minimalVersion: String       // обязательное поле
  anchorEvent: String?
  jokersPerWeek: Int           // 1–2 «жетона» легального пропуска, сгорают в Вс (не копятся)

Goal
  title, targetValue, unit, deadline
  milestones: [Milestone]      // этапы (напр. вес: шаги по 5 кг)
  linkedHabits: [Habit]        // трекается поведение, не результат
  horizon: .fiveYear / .year / .quarter

DayLog
  date, dayType
  energyMorning: Int?  energyEvening: Int?   // слайдер 1–5
  moodNote: String?

WeeklyReview / AnnualReview — см. раздел 7.
```

Правила статусов:
- `doneMini` визуально **идентичен** `doneFull`: одна и та же галочка, никаких серых чекбоксов и процентов (Гайз: «мини-версия и есть цель»).
- `noResource` («не было ресурса») — легальный нейтральный статус, не «провал».
- Слов «провал», «сорвался», «просрочено N дней» в UI не существует (А3, П17).

## 4. Ядро фич MVP

### 4.1 Quick capture (П12) — приоритет №1
Добавление задачи ≤2 тапа: виджет + кнопка на главном + Shortcut/Action Button. Единственное обязательное поле — текст. Всё падает в Inbox. Это протез рабочей памяти — фича, ради которой всё затевалось («всё забываю»).

### 4.2 Разбор Inbox
По **одной задаче за раз** (анти-overwhelm), кнопки: сегодня / в день X / мини-версией / в «когда-нибудь» / удалить / «вне моего влияния» (→ архив «отпустить»). Правило 2 минут действует только внутри сессии разбора (Противоречие 7).
«Когда-нибудь» автопротухает: элементы старше 60–90 дней предлагаются к пакетному удалению без чувства потери.

### 4.3 Экран Today
- Секции по сферам; якорная сверху со звездой.
- Кнопка «поковырять 5 минут» на любой задаче: таймер 5 мин → «продолжить / отложить с чистой совестью». Оба исхода = успех, статус `started` (П13).
- Начатые вчера задачи сортируются выше неначатых.
- Кнопка **«Плохой день»**: одно нажатие сворачивает день до 1 обязательного + рутины здоровья, только мини-версии.

### 4.4 Экран недели (П15)
Неделя первична, день производный. Метрики считаются по неделе — один нулевой день не портит показатель. «Больших камней» недели — не более 3, UI не даёт больше.

### 4.5 NagEngine — три класса напоминаний
```
.routine   → 1 push + badge. Не эскалирует.
.focus     → до 2 push/день, текст = выбор: «Делаю (таймер) / Мини / Перенести».
.critical  → эскалация каждые 15 мин до принятия решения.
             ЛИМИТ: max 2–3 задачи .critical в сутки, валидация в UI.
             Только: лекарства, жёсткие внешние дедлайны, встречи.
```
- Notification actions: «Сделано / Мини / +1 час» — реагируют без открытия приложения.
- Лимит iOS 64 pending-уведомления: перепланирование очереди при каждом foreground + BGAppRefreshTask.
- Правило двух пропусков: 1-й пропуск — тишина; 2-й подряд — мягкое предложение мини-версии, без красного.
- Правило переносов: `moveCount >= 3` → диалог «уменьшить до мини / удалить / поставить конкретный слот».
- Fresh start-пуши (класс routine): Пн утром / 1-го числа — «Чистая страница. Выбрать 1–3 вещи?»
- Защищённый слот (опц.): на выбранный час — тишина всего, кроме .critical.

### 4.6 Шлюз просрочки и Протокол амнистии — два режима
- **Обычный ритм** (пропуск ≤ 2 дней): при запуске `fullScreenCover` с задачами `deadline < now`, по каждой: Сделано / Мини / Перенести / Отменить. Пока не разобрано — дальше не пускает.
- **Амнистия** (не заходил 3+ дня): шлюз НЕ показывается. Экран: «Рад, что ты вернулся. Прошлое закрыто. Выберем 2 задачи на сегодня?» Хвосты автоматически уходят в скрытый архив (не удалены). Празднование возврата — усиленная анимация.
- Счётчик просрочки как публичное состояние **не существует** (А3).

### 4.7 Привычки (модуль TeymiaHabit + доработки)
- Обязательная мини-версия с валидацией bad day: «Сможешь в самый плохой день? Сомневаешься — уменьши».
- Джокеры: 1–2/неделю, списываются автоматически, сгорают в Вс.
- Якорь «после чего» + напоминание по событию-прокси, не только по часам (П6).
- Лимит: max 3 одновременно активных новых привычки; 4-я — только через архивацию одной.
- Диагностика после 4 пропусков — флоу P→A→M: «Замечал напоминание? → Слишком сложно? → Просто не хотелось (ок, бывает / пауза)». Ни одна ветка не упрекает.
- Перевыполнение — конфетти, но планка НЕ повышается автоматически (А5).

### 4.8 Линтер формулировок (через Claude API)
Задача без глагола / со словами «разобраться», «заняться», «подумать» → подсказка «Какое первое физическое действие?» + предложенная переформулировка (П8). Мысли/тревоги распознаются и предлагаются в Inbox-«парковку», не в задачи.

## 5. Метрики (что видит пользователь)

**Главная метрика: скорость возврата.** `recovery_time = первый день активности после пропуска − первый день пропуска`. Показ: «Вернулся за 1 день — это и есть навык» + медиана за месяц.

- Окно «N из 10» вместо стриков: выполнение привычки в скользящем окне (джокеры не считаются пропуском). Стрики не существуют нигде (А1).
- Энергия: слайдеры утро/вечер 1–5 → недельное среднее, тренд, корреляции (после чего проседает/растёт). Это операционализация цели «поднять батарейку до 70%».
- Индекс калибровки: план vs факт. При хроническом перекосе — «Ты планируешь из состояния супермена. Завтра лимит 3» + автопонижение дефолтного лимита.
- Экран «на что уходит неделя»: касания по сферам, без оценок. Достаточно видеть, что важные сферы были тронуты.
- Done list вечером — что сделано (включая мини и старты), вместо руминации о несделанном.
- Вес и подобные результат-метрики: фоновый график недельных средних, ежедневные точки не показываются. Ежедневно трекается только поведение.

## 6. Цели и горизонты — спрятанная вертикаль (А7, Противоречие 6)

Вертикаль существует в данных, но **не в ежедневном UI**. Горизонт видимого — неделя.

```
Направление (75 лет)  — 2–3 предложения текстом, видно только в годовое ревью
Пятилетка (2026–2031) — фазы: 1) восстановление + закрыть кредиты  2) капитал
Год                   — 1–3 цели (Goal)
Неделя                — до 3 «камней»  ← максимум видимого по умолчанию
День                  — лимит по типу дня
```

- Прогресс цели: разбивка на этапы (вес — по 5 кг). Отображение по small-area hypothesis: до 50% крупно «сделано», после 50% крупно «осталось». Переключение автоматическое.
- Помесячные планки нелинейные (лесенка с затуханием для веса).
- Мандала-чарт (8×8) — опциональный экран *постановки* цели (визуальная декомпозиция), не ежедневный интерфейс.

## 7. Ритуалы (все ≤5 мин, все опциональны, ни один не копит долг — А6)

- **Утро (2 мин):** тип дня → приложение предлагает 3 задачи из пула недели → «Да / Перенести». Выбор якорной через вопрос Келлера: «Что одно сделает остальное легче?»
- **Закрытие дня (3 мин):** done list → хвостам статус (кнопка «на завтра» доступна, но см. правило переносов) → до 3 задач на завтра → слайдер энергии.
- **Недельное ревью (5 мин, Вс):** guided-флоу 3 экрана: inbox пуст? / что протухло — амнистировать? / 1–3 камня на неделю. Пропуск ревью не создаёт долга.
- **Годовое ревью — 2 января:** что было / что дало плюс / что съело энергию / как изменилась жизнь / цели года.
- **Ревью дня рождения (13 июля):** персональный новый год. Первый запуск 2026 — стартовый замер: энергия, деньги, люди, здоровье.

## 8. Награды (А11: без накопительной геймификации)

- **Момент:** хаптика + анимация при каждой отметке; усиленная — за выполнение в плохой день и за возврат после пропуска. Награда за старт ≥ награды за финиш (П13).
- **Неделя:** маленькая реальная награда (не еда), условие — «вёл систему», не «был идеален».
- **Месяц:** крупная реальная награда (пример: мышка за 80k ₸). Условие: обработано ≥80% дней, **мини-версии, переносы и амнистии считаются** обработкой. Один срыв не обнуляет право на награду.
- Очков, уровней, лидербордов, замерзающих стриков — нет (А2, А10, А11).

## 9. Claude API внутри приложения

Endpoint `/v1/messages`, ключ пользователя в Keychain. Вызовы только по явному действию (планирование), не на каждый чих. Функции:
1. **Декомпозиция цели:** «прочитать книгу 500 стр» → проверка реализма → план (10 стр/день, 2 мес) → по подтверждению создаются Goal + Milestones + повторяющаяся задача.
2. **Проверка на реализм** («40 кг за неделю» → мягкий пуш-бэк с расчётом).
3. **Линтер формулировок** (4.8).
4. **Диалог разбора** («у меня задача X, не знаю как подступиться») → structured output → черновик плана в UI.
Ответы — JSON (structured output), парсинг с try/catch, деградация: нет сети/ключа → все функции просто скрыты, приложение полноценно без них.

## 10. Тон и копирайтинг (П17)

- Рамка «забота, не мораль» (Дэвис). Ошибки — ошибки дизайна, не характера: «Похоже, эта версия была великовата. Урежем?» — вместо «Вы пропустили 5 дней».
- Запрещённые слова в UI: провал, сорвался, ленился, просрочено, опять.
- Идентичность — только позитивно и через частоту («9 из 10 — ты человек, который возвращается»), никогда через пропуск (А8).
- Тексты уведомлений ротируются (не приедаются), тон — ассистент, не надзиратель.

## 11. Чего НЕ делать (краткий свод А1–А11)

## 12. План разработки по сессиям (Claude Code)

1. **Сессия 1 (Сб):** форк, сборка на iPhone (free provisioning), выпилить RevenueCat/пейволл, карта проекта. Промт готов.
2. **Сессия 2 (Вс, слот 17–19):** модель TaskItem + LifeArea + экран Today (без нага).
3. **Сессия 3:** NagEngine (3 класса, notification actions, лимит 64, правило двух пропусков и трёх переносов).
4. **Сессия 4:** шлюз просрочки + протокол амнистии.
5. **Сессия 5:** quick capture (виджет) + Inbox-разбор + линтер (пока без API — правила regex, API позже).
6. **Сессия 6:** типы дней + лимиты + якорная + «плохой день».
7. **Сессия 7:** ритуалы (утро/закрытие/неделя) + done list + слайдер энергии.
8. **Сессия 8:** метрики (возврат, N из 10, калибровка) + джокеры.
9. **Сессия 9:** Goal/Milestone + прогресс small-area + вес как недельный график.
10. **Сессия 10:** Claude API (декомпозиция, реализм, линтер v2).
Дальше: награды, мандала-экран, виджеты/Live Activities, защищённый слот.

Правило сессий: одна сессия = один модуль, минимальные диффы, в конце — обновление карты проекта. Обе базы знаний лежат в репо, Claude Code читает их в начале каждой сессии.

## 13. План внедрения — первые 30 дней (старт: Пн 07.07, на «Напоминаниях» Apple)

- **Неделя 1 — стабилизация:** 1 главная + ≤2 мелких + 3 рутины (вода, таблетки, вечерний вопрос «что переносим/удаляем»). Точка слома — день 4 («могу больше»). Нельзя.
- **Неделя 2 — контексты:** сферы + 3 фокуса недели (выбираются в Вс). Максимум +1 привычка.
- **Неделя 3 — кризис новизны:** первый долг. К этому моменту должна подоспеть своя сборка с амнистией; если нет — ручной разбор хвостов в Вс, без самокритики.
- **Неделя 4 — калибровка:** убрать игнорируемые уведомления и лишние рутины. Итог месяца — не новая жизнь, а система, которой доверяешь.
- Критерий успеха фазы 1: **60+ дней ведения и ≥1 возврат после срыва.**

## 14. «Потом» (не смотреть до конца фазы 1)

- Фаза 2: дать 2–3 близким людям, собрать обратную связь; блог документирует путь.
- Фаза 3 (2027, если продукт жив): App Store, бесплатно → локализация → цена (рынок утилит: $2–5/мес или разовая, не $10).
- Еженедельный чек ранних сигналов спада (5 вопросов: раздражительность 3+ дня, сон рушится, нет тяги к тому, что заряжало…) → предложение «красной недели».
- Пересборка соло-версии недельного шаблона (после расставания) — отдельный документ, вне этой спеки.

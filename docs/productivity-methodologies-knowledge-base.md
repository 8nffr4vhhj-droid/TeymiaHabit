# Методологии продуктивности, привычек и тайм-менеджмента
## База знаний для проектирования персонального планировщика

**Назначение документа:** справочник для AI-ассистента при разработке фич приложения-планировщика.
**Профиль целевого пользователя:** взрослый, черты СДВГ + депрессивный эпизод, нестабильная мотивация, полная занятость + подработки, история заброшенных трекеров по циклу «пропустил → вина → бросил».
**Концепция приложения:** «система для плохих дней» — минимальные версии задач, амнистия просрочки, отказ от стриков, метрика скорости возврата, три класса напоминаний (критичное / полезное / опциональное), лимиты задач по типам дней.

**Обозначения в тексте:**
- ✅ — применимо для профиля «СДВГ + депрессия + низкий ресурс»
- ⚠️ — применимо с модификацией
- ❌ — противопоказано
- → App: — конкретная механика для приложения (экран / поле / алгоритм / правило)

# ЧАСТЬ I. РАЗБОР СИСТЕМ

## 1. Getting Things Done — Дэвид Аллен (2001)

### а) Механики
1. **Capture (сбор):** любая мысль/обязательство немедленно выгружается во внешний inbox, а не держится в голове. Inbox один (или минимум точек входа), доступен за <5 секунд.
2. **Clarify (прояснение):** каждый элемент inbox обрабатывается вопросами: «Это действие? Одно действие или проект? Какое *физическое следующее действие*?» Формулировка задачи обязана начинаться с глагола конкретного действия («позвонить X», а не «разобраться с налогами»).
3. **Правило 2 минут:** если действие занимает <2 минут — сделать сразу при обработке, не записывать.
4. **Organize:** действия раскладываются по контекстам (@телефон, @компьютер, @магазин), проекты — в отдельный список, несрочное — в Someday/Maybe, ожидания от других — в Waiting For.
5. **Календарь только для «жёсткого ландшафта»:** в календарь попадает только то, что привязано к дате/времени объективно. Списки дел в календарь не пишутся.
6. **Weekly Review:** еженедельный ритуал ~1–2 часа: очистить inbox, пройти все списки, обновить проекты, просмотреть Someday/Maybe.
7. **Выбор задачи в моменте** по 4 критериям: контекст → доступное время → доступная энергия → приоритет. Энергия — легитимный фильтр: «нет сил» = выбирай задачу под низкую энергию, а не «заставь себя».
8. **Проект = любой результат, требующий >1 действия.** У каждого активного проекта всегда должно быть определено следующее действие.
9. **Mind Sweep:** периодическая полная выгрузка головы по списку триггеров (работа, дом, финансы, здоровье…).

### б) Применимость к профилю
- ✅ Capture — критично для СДВГ: рабочая память ненадёжна, внешняя система обязательна. Это самая ценная механика GTD для профиля.
- ✅ «Следующее физическое действие» — напрямую бьёт по прокрастинации из-за аморфности задачи (основной триггер и при СДВГ, и при депрессии).
- ✅ Фильтр по энергии (п.7) — редкая для 2001 года легитимизация низкого ресурса; идейный предок «минимальных версий задач».
- ⚠️ Правило 2 минут — полезно, но при СДВГ провоцирует цепочку «сделаю сразу» → провалился в побочную задачу на час. Модификация: правило применять только в выделенной сессии разбора inbox.
- ❌ Полная система (6 списков, контексты, weekly review 2 часа) — избыточна. GTD требует высокой исполнительной функции для *поддержания самой системы*; классический паттерн СДВГ — неделя энтузиазма, потом система умирает и добавляет вины. Weekly review как обязательный 2-часовой ритуал почти гарантированно сорвётся.
- ❌ Someday/Maybe как открытый бесконечный список — превращается в кладбище вины.

### в) Перенос в приложение
- → App: **Quick capture** — добавление задачи с главного экрана/виджета/шорткатом за ≤2 тапа, без обязательных полей кроме текста. Все атрибуты (дата, тип, проект) — опциональны и проставляются позже.
- → App: **Inbox как отдельный статус** задач «неразобрано»; экран разбора показывает по одной задаче за раз (анти-overwhelm) с кнопками: «сделать мини-версией», «в день X», «в когда-нибудь», «удалить».
- → App: **Линтер формулировок** — если задача не начинается с глагола / длиннее N слов / содержит «разобраться», «заняться», подсказка: «Какое первое физическое действие?» + предложение переформулировать (можно LLM-ом).
- → App: **Поле «энергия» у задачи** (низкая/средняя/высокая) и фильтр «покажи, что можно сделать сейчас при низкой энергии» — прямой вход в режим плохого дня.
- → App: **Weekly review в микроформате** — не 2 часа, а 5-минутный guided-флоу: 3 экрана (inbox пуст? / что протухло — амнистировать? / 1–3 задачи на неделю). Пропуск ревью не создаёт долга.
- → App: **Someday с автопротуханием** — элементы старше 60–90 дней предлагаются к удалению пачкой, без чувства потери («не трогал 3 месяца — видимо, не нужно. Удалить 12 штук?»).

## 2. The 7 Habits of Highly Effective People — Стивен Кови (1989)

### а) Механики
1. **Проактивность:** фокус на «круге влияния» (что могу изменить), а не «круге забот» (что меня тревожит, но неподконтрольно).
2. **Begin with the End in Mind:** личная миссия/видение → из неё выводятся роли и цели.
3. **Матрица Эйзенхауэра (Q1–Q4):** срочное/важное. Ключевой тезис: жить надо в Q2 (важное несрочное — профилактика, отношения, здоровье), а не в Q1 (тушение пожаров) и Q3 (чужая срочность).
4. **Планирование от ролей:** неделя планируется по ролям (работник, партнёр, я-сам), в каждой роли 1–2 «больших камня» (big rocks first — сначала важное, потом мелочь заполняет промежутки).
5. **Недельный, а не дневной горизонт планирования:** день слишком волатилен, неделя даёт гибкость сдвинуть, не сорвав.
6. **P/PC balance:** результат (Production) не должен разрушать ресурс, который его производит (Production Capability). «Заточка пилы» — регулярное восстановление (тело, сон, отношения) — не награда, а часть системы.
7. **Win/Win и делегирование результата, а не метода** — межличностные привычки (для соло-приложения второстепенны).

### б) Применимость к профилю
- ✅ Недельный горизонт вместо дневного — снижает цену сорванного дня: план недели переживает 1–2 плохих дня без «всё пропало».
- ✅ P/PC-баланс — прямое обоснование того, что отдых/сон/восстановление — это задачи первого класса, а не то, что «после дел». Для депрессии критично.
- ✅ Круг влияния — полезный фильтр против руминации; операционализируется как вопрос при разборе тревожных «задач».
- ⚠️ Матрица Эйзенхауэра — концептуально верна, но как ежедневный инструмент сортировки требует слишком много решений (decision fatigue). Годится как редкий аудит, не как интерфейс.
- ❌ Личная миссия, видение на десятилетия, планирование от жизненных ролей — при депрессии вопросы «в чём смысл моей жизни» контрпродуктивны и могут утяжелять состояние. Горизонт планирования должен быть коротким.
- ❌ «Big rocks first» в лоб (начни день с самого важного) конфликтует с СДВГ-механикой разгона — см. противоречие №2 в Синтезе.

### в) Перенос в приложение
- → App: **Планирование единицей «неделя»**: экран недели первичен, день — производный. Метрика выполнения считается по неделе, не по дню → один нулевой день не портит показатель.
- → App: **Категория задач «восстановление»** (сон, прогулка, еда) — учитывается в дневном лимите как полноценные задачи, на плохой день подставляется автоматически.
- → App: **Ограничение «больших камней»**: не более 1–3 задач недели с меткой «важное»; UI физически не даёт назначить больше (проверенный паттерн против перегруза).
- → App: при разборе inbox — кнопка «это вне моего влияния» → задача конвертируется в запись «отпустить» и архивируется, а не висит.

## 3. Atomic Habits — Джеймс Клир (2018)

### а) Механики
1. **Четыре закона поведения:** сделать очевидным (cue) / привлекательным (craving) / лёгким (response) / удовлетворяющим (reward). Инверсия — для отучения от плохих привычек.
2. **Правило 2 минут:** новая привычка масштабируется вниз до действия ≤2 минут («читать» → «открыть книгу»). Сначала закрепляется факт появления, потом объём.
3. **Habit stacking:** привязка новой привычки к существующей: «После [текущая привычка] я сделаю [новая]». Якорь — событие, а не время.
4. **Environment design:** изменить среду, а не полагаться на силу воли: убрать трение для нужного (кроссовки у двери), добавить трение для ненужного (телефон в другой комнате).
5. **Identity-based habits:** цель — не «пробежать марафон», а «стать бегуном»; каждое выполнение = голос за идентичность.
6. **Habit tracking:** визуальная отметка выполнения как награда; «don't break the chain».
7. **Правило «never miss twice»:** один пропуск — случайность, два подряд — начало новой (плохой) привычки. Вернуться на следующий раз важнее, чем не пропускать.
8. **Goldilocks rule:** удержание сложности на грани посильного — слишком легко скучно, слишком трудно бросается.
9. **Среда > мотивация, система > цель:** «You do not rise to the level of your goals. You fall to the level of your systems».

### б) Применимость к профилю
- ✅ Правило 2 минут — ядро концепции «минимальной версии задачи». Прямое совпадение с дизайном приложения.
- ✅ Never miss twice — это почти дословно «метрика скорости возврата»: важен не стрик, а быстрота возвращения. Самая недооценённая механика книги.
- ✅ Environment design — для СДВГ работает лучше всех «мотивационных» техник, т.к. не требует исполнительной функции в моменте.
- ✅ Habit stacking — якорение к событию, а не ко времени, идеально для СДВГ (слепота ко времени, time blindness).
- ⚠️ Identity-based habits — при депрессии опасен обратный ход: «я пропустил → это голос за то, что я ничтожество». Применять только в позитивной формулировке и никогда не показывать пользователю негативную интерпретацию пропуска.
- ❌ «Don't break the chain» / стрики — главный источник цикла «пропустил → вина → бросил». Клир сам противоречит себе: стрик-механика (п.6) конфликтует с never miss twice (п.7). Для профиля выбираем п.7, отбрасываем п.6.
- ❌ Goldilocks rule в автоматике — алгоритмическое «пора усложнить!» на плохой неделе вредно; усложнение только по явному согласию пользователя.

### в) Перенос в приложение
- → App: **Обязательное поле «минимальная версия»** у повторяющихся задач: создавая привычку «зарядка 20 мин», пользователь тут же задаёт мини-версию («1 упражнение»). В плохой день показывается только мини-версия. Выполнение мини = полноценное выполнение (без «полузачёта» и серых галочек).
- → App: **Метрика возврата вместо стрика:** показывать «вернулся через 1 день» / медиану скорости возврата за месяц. Формула: `recovery_time = дата_первого_выполнения_после_пропуска − дата_первого_пропуска`. Празднуется возврат, не непрерывность: «Ты вернулся за 2 дня — это и есть навык».
- → App: **Якорь привычки — поле «после чего»** (после кофе, после прихода домой) вместо/в дополнение к времени; напоминание можно триггерить по геозоне/времени-прокси якоря.
- → App: **Подсказки по среде** в карточке привычки: поле «что подготовить заранее» с вечерним напоминанием класса «опциональное» («выложи кроссовки»).
- → App: **Правило двух пропусков в логике эскалации:** первый пропуск — тишина (норма), второй подряд — мягкое предложение мини-версии («сегодня только открыть книгу?»), не тревога и не красный цвет.

## 4. Tiny Habits — Би Джей Фогг (2019)

### а) Механики
1. **Модель поведения B = MAP:** поведение случается, когда одновременно сходятся Motivation + Ability + Prompt. Мотивация — самый ненадёжный компонент; проектировать надо Ability (упрощать) и Prompt (триггер), а не накачивать мотивацию.
2. **Рецепт привычки:** «После того как я [якорь], я сделаю [крошечное поведение]». Крошечное = 30 секунд или меньше (2 отжимания, 1 зуб почистить нитью).
3. **Celebration (празднование):** немедленная искусственная позитивная эмоция после выполнения (сказать «Yes!», улыбнуться, жест). Фогг: привычку закрепляет эмоция, а не повторение. «Emotions create habits».
4. **Motivation wave:** мотивация приходит волнами; на пике волны — не делать больше, а *настраивать среду и систему* на будущее (купить, установить, подготовить), потому что пик пройдёт.
5. **Troubleshooting порядок:** если поведение не происходит — сначала проверь Prompt (был ли триггер?), потом Ability (слишком сложно? упрости), и только в конце Motivation. Никогда не начинай с «мне не хватает дисциплины».
6. **Pearl habits:** привычки, якорем которых служит раздражитель (после гудка пробки — глубокий вдох): превращение негативного триггера в позитивный.
7. **Rehearsal:** физическая репетиция связки якорь→действие 7–10 раз подряд как ускоренная тренировка нейронной связи.
8. **Никакой самокритики:** ошибки дизайна поведения — это ошибки *дизайна*, не характера. Итерация рецепта вместо самообвинения.

### б) Применимость к профилю
- ✅ B=MAP — фундаментальная модель для всего приложения: каждый фейл пользователя интерпретируется как «упростить (A) или починить триггер (P)», никогда как «слабая воля (M)». Это философия «системы для плохих дней» в одной формуле.
- ✅ Motivation wave — объясняет и цикл заброшенных трекеров (настройка на пике, использование в яме), и что делать: на пике мотивации пользователь должен *настраивать мини-версии и амнистии*, а не набирать 15 привычек.
- ✅ Celebration — при депрессии ослаблен внутренний reward-сигнал; внешнее празднование частично компенсирует. Для СДВГ немедленность награды критична (отложенная награда не работает).
- ✅ Troubleshooting-порядок P→A→M — готовый алгоритм диалога после серии пропусков.
- ⚠️ «30 секунд» — иногда слишком радикально для рабочих задач; для привычек — да, для задач — мини-версия может быть 2–5 минут.
- ❌ Противопоказаний практически нет — это самая безопасная система списка для данного профиля.

### в) Перенос в приложение
- → App: **Диагностический флоу после пропусков** (алгоритм P→A→M): «Привычка X пропущена 4 раза. Ты замечал напоминание? [нет → сменить триггер/время] Было слишком сложно? [да → урезать мини-версию ещё] Просто не хотелось? [ок, бывает — оставить как есть / поставить на паузу]». Ни одна ветка не содержит упрёка.
- → App: **Микро-celebration в UI:** анимация/хаптика/звук немедленно при отметке выполнения; усиленная — при выполнении в плохой день и при возврате после пропуска. Награда за возврат ≥ награды за регулярность.
- → App: **Онбординг на пике мотивации:** при регистрации (пользователь мотивирован) заставить настроить именно инфраструктуру плохих дней: мини-версии, лимиты, тон уведомлений. Прямо объяснить: «Ты настраиваешь систему для себя-уставшего».
- → App: **Жёсткий лимит новых привычек:** не более 3 одновременно активных новых привычек; добавление 4-й требует архивировать одну или подтвердить осознанно.
- → App: **Копирайтинг всех состояний ошибки/пропуска** по принципу «ошибка дизайна, не характера»: «Похоже, эта версия задачи была великовата. Урежем?» — вместо «Вы пропустили 5 дней».

## 5. Mini Habits — Стивен Гайз (2013)

### а) Механики
1. **Stupid small:** требование к привычке настолько маленькое, что провалить невозможно даже в худший день: 1 отжимание, 50 слов, 2 страницы. Планка — по худшему дню, не по среднему.
2. **Требование ≠ ожидание:** цель на бумаге — 1 отжимание; всё сверх — бонус. Запрещено втихую поднимать внутреннюю планку («на самом деле я жду от себя 30») — это разрушает механику.
3. **Эффект начала:** главное сопротивление — старт; после первого отжимания продолжить легко (физика инерции поведения). Мини-требование — это трюк для гарантированного старта.
4. **Сила воли вместо мотивации:** мотивация ненадёжна, но на «stupid small» хватает даже минимальной воли. Расход воли ≈ 0 → нет истощения → нет срывов.
5. **Не более 2–4 мини-привычек одновременно.**
6. **Проверка на bad day:** перед добавлением привычки — вопрос «смогу ли я это в самый плохой день?». Если нет — уменьшить.
7. **Против «стретч-целей»:** большие цели дают всплеск мотивации при постановке и хроническую вину при исполнении.

### б) Применимость к профилю
- ✅ Вся книга — это концентрат идеологии «системы для плохих дней». Планка по худшему дню — готовый принцип калибровки мини-версий.
- ✅ «Требование ≠ ожидание» — тонкий, но важный пункт: приложение не должно нигде показывать «полную» версию как «настоящую цель», иначе выполнение мини ощущается как поражение.
- ✅ Эффект начала — обоснование, почему выполнение мини-версии засчитывается на 100%: цель мини-версии — старт, а старт статистически тянет за собой больше.
- ⚠️ «Сила воли» как термин устарел (ego depletion плохо реплицируется), но практический вывод (маленькое действие не требует ресурса) верен.
- ❌ Противопоказаний нет.

### в) Перенос в приложение
- → App: **Валидация мини-версии вопросом bad day:** при создании мини-версии — подсказка «Сможешь сделать это в день, когда всё плохо? Если сомневаешься — уменьши ещё».
- → App: **Мини-версия и есть цель:** в UI плохого дня полная версия вообще не отображается. Галочка за мини визуально идентична галочке за полную (никаких половинчатых чекбоксов, процентов, серых цветов).
- → App: **Перевыполнение фиксируется как приятный факт** («сделал 20 мин вместо 2» — маленький конфетти), но НЕ повышает планку автоматически и не пересчитывает «норму».

## 6. The Power of Habit — Чарльз Дахигг (2012)

### а) Механики
1. **Петля привычки:** Cue → Routine → Reward. Привычка = автоматизм, закодированный в базальных ганглиях; сознание в петле почти не участвует.
2. **Golden rule изменения привычки:** нельзя удалить привычку — можно заменить Routine, сохранив Cue и Reward. Алгоритм: определи рутину → поэкспериментируй с наградами (чего на самом деле хочется?) → изолируй триггер (5 категорий: место, время, эмоция, люди, предыдущее действие) → составь план замены.
3. **Craving:** привычку держит предвкушение награды; без сформированного craving петля не автоматизируется.
4. **Keystone habits:** некоторые привычки (спорт, сон, семейный ужин) вызывают каскад изменений в других областях без прямой работы над ними.
5. **Small wins:** цепочки маленьких побед создают импульс для структурных изменений.
6. **Implementation intentions / планирование болевых точек:** заранее прописанный план «если случится X — сделаю Y» резко повышает выживаемость привычки в стрессовой ситуации.

### б) Применимость к профилю
- ✅ Диагностика триггера по 5 категориям — готовая форма для разбора «почему привычка не заходит» и «почему срываюсь в думскролл».
- ✅ Keystone habits — обоснование приоритета сна/движения в дефолтных шаблонах: при депрессии сон и активность — терапевтические мишени первой линии.
- ✅ Small wins — совпадает с мини-версиями и празднованием возврата.
- ⚠️ Golden rule (замена рутины) — рабочая техника, но требует самонаблюдения в моменте; при СДВГ нужна внешняя поддержка (напоминание-вопрос), сама по себе не взлетит.
- ❌ Прямых противопоказаний нет; книга скорее объяснительная, чем предписывающая.

### в) Перенос в приложение
- → App: **Поле «триггер» у привычки** с 5 категориями Дахигга (место/время/эмоция/люди/после чего) — используется и для напоминаний, и для диагностики при пропусках.
- → App: **Шаблоны keystone-привычек** в онбординге: сон, выход на улицу, приём пищи — предлагаются первыми, с готовыми мини-версиями.
- → App: **План «если-то» у важных привычек:** опциональное поле «Если [срыв-ситуация], то [запасное действие]», показывается в напоминании соответствующего контекста.

## 7. Deep Work — Кэл Ньюпорт (2016)

### а) Механики
1. **Deep work:** когнитивно требовательная работа без отвлечений создаёт непропорционально больше ценности; способность к ней — тренируемый навык и дефицитный ресурс.
2. **Лимит глубокой работы:** даже у тренированных — максимум ~4 часа в день. Планировать больше — самообман.
3. **Четыре режима:** monastic (полная изоляция), bimodal (дни/недели глубины), rhythmic (ежедневный фиксированный слот — рекомендуется большинству), journalistic (впрыгивать в любую щель — только для опытных).
4. **Ритуалы входа:** фиксированное место, время, правила («кофе, наушники, телефон в ящик») снижают цену входа в фокус.
5. **Time blocking:** каждый час рабочего дня получает назначение заранее; при сломе плана — пересборка блоков, а не отказ от плана.
6. **Shutdown ritual:** явное завершение рабочего дня с фразой-маркером; после — никакой работы. Незакрытые циклы фиксируются в план завтра (разгрузка эффекта Зейгарник).
7. **Attention residue:** переключение задач оставляет «осадок внимания», снижающий качество следующей задачи на десятки минут → батчинг мелочи, а не interleaving.
8. **Embrace boredom:** тренировка терпимости к скуке (не доставать телефон в очереди) как тренажёр фокуса.
9. **Drain the shallows:** мелочь (почта, созвоны) сжимается квотами и батчится, освобождая место глубине.

### б) Применимость к профилю
- ✅ Лимит 4 часа — обоснование лимитов задач по типам дней: даже у здорового человека глубокого ресурса мало; при СДВГ+депрессии — меньше. Плановая ёмкость дня должна быть унизительно маленькой и это норма.
- ✅ Ритуалы входа — при СДВГ проблема инициации задачи центральная; скриптованный вход («открой файл, поставь таймер») снижает стоимость старта.
- ✅ Shutdown ritual — при депрессии руминация о несделанном вечером мешает сну; ритуал закрытия дня с переносом хвостов — противоруминационная механика.
- ✅ Attention residue — аргумент для UI: не показывать список остальных задач во время выполнения текущей.
- ⚠️ Time blocking всего дня — при СДВГ жёсткая почасовая сетка ломается к 11:00 и генерирует вину. Модификация: блоки только для 1–2 якорных слотов, остальное — гибкий список.
- ❌ Monastic/bimodal режимы — несовместимы с полной занятостью + подработками.
- ❌ Embrace boredom как тренировка — благое пожелание, не операционализируется в приложении задач; при СДВГ «просто терпи скуку» без поддержки не работает.

### в) Перенос в приложение
- → App: **Фокус-режим выполнения задачи:** полноэкранная карточка текущей задачи, таймер, остальной список скрыт. Опция «шаг за шагом» — показывать только текущий подшаг.
- → App: **Ритуал входа как чеклист** в карточке тяжёлой задачи (3 пункта, настраивается один раз, выполняется тапами) — снижает стоимость инициации.
- → App: **Вечернее закрытие дня** (2 минуты, напоминание класса «полезное»): непросмотренные задачи автоматически уезжают в завтра/неделю без красных меток; экран говорит «День закрыт. Хвосты перенесены, это нормально».
- → App: **Лимит «глубоких» задач: 1 в день** (метка «требует фокуса»); планирование второй — предупреждение «даже у Ньюпорта лимит 4 часа».

## 8. Hyperfocus — Крис Бэйли (2018)

### а) Механики
1. **Внимание — ограниченное пространство (attentional space):** одновременно удерживается ~4 объекта; сложная задача занимает почти всё пространство. Планирование = управление этим пространством, не временем.
2. **Hyperfocus (осознанный):** выбрать один объект → устранить отвлечения заранее → удерживать → возвращать внимание при уплывании. Возвращение — норма процесса, а не сбой.
3. **Scatterfocus (рассеянный режим):** намеренное блуждание ума (прогулка, душ) для консолидации, планирования и креативных связок. Отдых мозга — продуктивный режим, а не потерянное время.
4. **Правило трёх:** утром выбрать 3 намерения на день (не 10). Вечером — сверка. Три — потому что помещается в рабочую память.
5. **Уровни отвлечений:** контролируемые/неконтролируемые × раздражающие/приятные — разные стратегии для каждого квадранта; приятные контролируемые (соцсети) убираются заранее, средой.
6. **Awareness chime:** периодический сигнал-вопрос «на что сейчас направлено моё внимание?» — тренировка мета-осознанности.
7. **Учёт энергии:** пики/спады энергии в течение дня — тяжёлое в пик (Biological Prime Time), механическое в спад.

### б) Применимость к профилю
- ✅ Правило трёх — готовый дефолт лимита обычного дня. Число подтверждено практикой многих систем (см. Синтез, принцип №2).
- ✅ «Возвращение внимания — норма» — та же философия, что метрика скорости возврата, но на микроуровне минут. Единый принцип на обоих масштабах — сильная концепция для приложения.
- ✅ Scatterfocus — легитимизация «ничегонеделания» как режима; при депрессии снимает вину за отдых.
- ✅ Biological Prime Time — при СДВГ колебания энергии выражены; поле энергии задачи + знание своего пика = лучший матчинг.
- ⚠️ Awareness chime — для СДВГ может стать ещё одним раздражающим уведомлением; только opt-in.
- ❌ Существенных противопоказаний нет.

### в) Перенос в приложение
- → App: **Дефолтный лимит обычного дня = 3 задачи** (+ настройка по типам дней: плохой 1, обычный 3, сильный 5). Попытка добавить сверх лимита → «Куда переносим лишнее?» вместо молчаливого разрастания списка.
- → App: **Утренний выбор 3 намерений** как отдельный 30-секундный флоу (напоминание класса «полезное»); пропуск утреннего выбора → день просто наследует план недели, без пустого экрана.
- → App: **Тип дня и Prime Time:** пользователь одним тапом задаёт тип дня (или приложение предлагает по косвенным сигналам — время первой активности, явный ответ на «как ты?»); сортировка задач подстраивается: в пик — фокусная, в яму — механические/мини.

## 9. The One Thing — Гэри Келлер, Джей Папасан (2013)

### а) Механики
1. **Фокусирующий вопрос:** «Какое ОДНО дело я могу сделать, благодаря которому всё остальное станет проще или ненужным?» — применяется на каждом горизонте (год → месяц → неделя → сегодня).
2. **Goal setting to the Now:** цепочка от долгосрочной цели вниз до «что сделать прямо сейчас» — связывает горизонты.
3. **Домино-эффект:** последовательное опрокидывание маленьких костяшек важнее параллельных усилий; одна правильная задача запускает каскад.
4. **Time block на One Thing:** до 4 часов утром на главное, до того как мир начнёт дёргать; блок защищается как встреча с врачом.
5. **Против мультизадачности и против «списков дел»:** длинный to-do — это «список выживания»; вместо него — success list из одного пункта.
6. **Willpower is depletable:** главное — на утро, пока ресурс есть.
7. **Баланс — миф; вместо него counterbalancing:** периоды осознанного перекоса с возвратом.

### б) Применимость к профилю
- ✅ Фокусирующий вопрос — отличный алгоритм для интерфейса выбора «якорной задачи дня»: помогает выбрать 1 из 3, когда парализует даже короткий список.
- ✅ Идея «одна задача = день прожит не зря» — правильная планка для плохого дня.
- ⚠️ 4-часовой утренний блок — нереален при полной занятости + подработках; сжимается до «одного защищённого слота 30–60 мин».
- ❌ Goal setting to the Now (постоянная сверка с целью на 5 лет) — при депрессии длинные горизонты давят; цепочку целей прятать, показывать только ближний конец.
- ❌ «Всё, кроме одного, неважно» в чистом виде — у пользователя есть обязательства (работа+подработки), которые не исчезнут от расстановки приоритетов; риск вины за «неправильную жизнь».

### в) Перенос в приложение
- → App: **Якорная задача дня:** среди дневных задач одна помечается звездой «если сделаю только её — день зачтён». Выполнена якорная → день визуально «успешный», сколько бы ни осталось.
- → App: **Помощник выбора якорной** — вопрос Келлера в UI: «Что одно сделает остальное легче или ненужным?» (показывается при выборе, кандидатов ≤3).
- → App: **Один защищённый слот** в день (опционально): интеграция с календарём, на это время — блокировка уведомлений приложения, кроме класса «критичное».

## 10. Essentialism — Грег МакКеон (2014)

### а) Механики
1. **Дисциплинированное преследование меньшего:** по умолчанию — «нет»; «да» — только осознанное исключение. Если не «hell yes», то «нет».
2. **Правило 90%:** оценивая опцию, ставь оценку 0–100; всё ниже 90 = 0 = отказ. Устраняет «неплохие» варианты, съедающие жизнь.
3. **Trade-off явный:** сказать «да» одному = сказать «нет» другому; вопрос не «как успеть всё», а «чем жертвую».
4. **Buffer:** закладывать 50% запаса времени на всё; планирование по лучшему сценарию — систематическая ошибка.
5. **Устранение препятствий:** найти одно «узкое место» процесса и убрать его, вместо того чтобы толкать сильнее.
6. **Progress = маленькие победы:** начинать с минимального жизнеспособного прогресса, «done is better than perfect».
7. **Routine as freedom:** рутина освобождает исполнительный ресурс для важного.
8. **Graceful no:** техники отказа без сжигания отношений (пауза, «дай проверю календарь», встречное предложение).

### б) Применимость к профилю
- ✅ Buffer 50% — при СДВГ систематическая недооценка времени (планирование по «идеальному себе») — один из главных источников просрочки. Автоматический буфер лечит это на уровне алгоритма.
- ✅ Явный trade-off — основа UX лимитов: добавляешь 4-ю задачу → выбери, какая из трёх уходит.
- ✅ «Меньше, но лучше» — общая рамка, совпадающая с лимитами по типам дней.
- ⚠️ Правило 90% — для решений о новых обязательствах хорошо; к бытовым задачам неприменимо (посуда не наберёт 90, но её надо мыть).
- ❌ «Скажи нет почти всему» — пользователь с подработками не может отказаться от обязательств; давление «настоящий эссенциалист бы ушёл» → вина. Рамку применять к *новым* добровольным обязательствам, не к существующим.

### в) Перенос в приложение
- → App: **Zero-sum планирование дня:** добавление задачи сверх лимита требует выбора «что убрать/перенести» — drag-свап, а не бесконечный список.
- → App: **Автобуфер оценок времени:** если у задачи есть оценка длительности, при планировании дня она умножается на 1.5 (коэффициент настраиваемый; после накопления статистики — персональный, из фактических данных «оценка vs факт»).
- → App: **Экран «на что уходит неделя»:** раз в неделю — пассивная сводка «выполнено по сферам» без оценочных суждений; помогает увидеть trade-off постфактум.

## 11. Eat That Frog! — Брайан Трейси (2001)

### а) Механики
1. **«Съешь лягушку» первой:** самое неприятное/важное дело — первым делом утра, без раздумий; если лягушки две — сначала более уродливая.
2. **Правило 80/20:** 20% задач дают 80% результата; научиться опознавать и защищать эти 20%.
3. **Метод ABCDE:** каждой задаче буква (A — серьёзные последствия, B — мягкие, C — приятно, D — делегировать, E — удалить); нельзя браться за B, пока есть A.
4. **Нарезка задач:** «salami slicing» (тонкие ломтики большой задачи) и «swiss cheese» (продырявить задачу короткими 5–10-минутными подходами).
5. **Single handling:** взял задачу — доведи до конца без переключений.
6. **Планирование с вечера:** список на завтра пишется накануне — подсознание работает ночью, утро стартует без решений.
7. **Закон трёх:** три задачи дают большую часть ценности дня.

### б) Применимость к профилю
- ✅ Нарезка (salami/swiss cheese) — прямой аналог декомпозиции и мини-подходов; «5 минут ковыряния» — легальный способ начать страшную задачу.
- ✅ Планирование с вечера — убирает утреннее решение «с чего начать» (утро при депрессии — худшее время для решений).
- ✅ Закон трёх — ещё одно подтверждение лимита «3».
- ❌ «Лягушка первой» — для профиля скорее вредно: при СДВГ+депрессии старт дня с самого тяжёлого часто = не стартовать вообще. Профилю нужен разгон от лёгкого к тяжёлому (см. Противоречие №2). Лягушку — на пик энергии, не на утро по умолчанию.
- ❌ ABCDE — пять градаций приоритета = decision fatigue; для профиля достаточно бинарного «якорная / остальное».
- ⚠️ Single handling — верно как идеал, но жёсткое требование неисполнимо при СДВГ; вместо запрета переключений — мягкий возврат («ты начинал X, вернёмся?»).

### в) Перенос в приложение
- → App: **Кнопка «поковырять 5 минут»** на любой страшной задаче: таймер на 5 минут, после — «продолжить / отложить с чистой совестью». Оба исхода = успех, задача получает отметку «начата».
- → App: **Вечернее планирование** в составе ритуала закрытия дня: «выбери до 3 задач на завтра» из недельного пула (напоминание класса «полезное»).
- → App: **Незавершённые начатые задачи** поднимаются в сортировке следующего дня выше неначатых (single handling мягко, через порядок, а не запреты).

## 12. Pomodoro Technique — Франческо Чирилло (1980-е/2006)

### а) Механики
1. **Таймер 25 минут** = 1 помидор; работа только над одной задачей; помидор неделим.
2. **Перерыв 5 минут** после каждого помидора; после 4 — длинный 15–30 минут.
3. **Внутренние отвлечения** («а проверю-ка почту») записываются в лист и откладываются, работа продолжается.
4. **Внешние прерывания:** стратегия «сообщить-договориться-перезвонить» или пометка; если прерывание неизбежно — помидор аннулируется (не бывает «пол-помидора»).
5. **Оценка задач в помидорах** и сравнение с фактом — калибровка чувства времени.
6. **Задача <1 помидора** — комбинируется с другими; >5–7 помидоров — разбивается.

### б) Применимость к профилю
- ✅ Таймер — внешняя структура времени, компенсирующая time blindness при СДВГ; звук конца = внешний «выключатель» гиперфокуса, звук начала = внешний инициатор.
- ✅ Лист отвлечений — техника парковки импульсов, крайне полезна при СДВГ: импульс записан → мозг отпускает.
- ✅ Оценка в помидорах — данные для персонального коэффициента буфера (см. Essentialism).
- ⚠️ 25 минут — не догма: при СДВГ рабочи и 10–15 мин (порог входа ниже), и 50 (если пошёл гиперфокус, обрывать больно). Длина должна быть настраиваемой, и «продлить без перерыва» — легальная кнопка.
- ❌ «Помидор неделим, прерванный аннулируется» — пуризм, создающий вину за прерывания, которые при СДВГ неизбежны. 12 минут фокуса из 25 — это 12 минут фокуса, а не ноль.

### в) Перенос в приложение
- → App: **Встроенный фокус-таймер** в карточке задачи: пресеты 10/15/25/45, продление в один тап, всё время фокуса записывается в лог задачи (никаких «сгоревших» интервалов).
- → App: **Кнопка «припарковать мысль»** во время фокус-режима: быстрый ввод → уходит в inbox, экран фокуса не покидается.
- → App: **Пассивная калибровка времени:** факт из таймера сравнивается с оценкой задачи; раз в месяц — инсайт «твои задачи занимают в среднем ×1.8 от оценки — учитываю это в планировании» (алгоритм буфера становится персональным).

## 13. Make Time — Джейк Кнапп, Джон Зерацки (2018)

### а) Механики
1. **Highlight:** каждый день выбирается один «хайлайт» (60–90 мин) по одному из критериев: срочность / удовлетворение / радость. Радость — легитимный критерий выбора главного дела.
2. **Laser:** тактики защиты внимания от «infinity pools» (бесконечных лент): разлогиниться, убрать иконки, «distraction-free phone», барьеры трения.
3. **Energize:** энергия тела — фундамент фокуса: движение, еда, кофеин по науке, тишина, сон. Тело заряжает батарею, от которой работают Highlight и Laser.
4. **Reflect:** вечером — 30 секунд: что сработало, что скорректировать завтра. Система = ежедневный микроэксперимент над собой, а не программа с «правильной» версией.
5. **83 тактики как меню:** авторы прямо говорят — не пытайтесь применять всё; выберите пару, тестируйте, выбрасывайте без сожалений.
6. **«Everything is a hot mess and that's OK»:** сбои — часть системы, дизайн важнее дисциплины.

### б) Применимость к профилю
- ✅ Тон книги — наименее виноватящий в жанре; рамка «эксперимент, а не экзамен» идеальна для истории заброшенных трекеров: нельзя «провалить» эксперимент.
- ✅ Highlight с критерием «радость» — при депрессии план из одних обязанностей ускоряет истощение; выбор якорной задачи «потому что хочется» должен быть легален и поощряем.
- ✅ Energize — телесные привычки как инфраструктура; совпадает с keystone habits Дахигга.
- ✅ Меню тактик вместо системы — модель для настроек приложения: включаемые модули, а не монолит.
- ❌ Противопоказаний практически нет.

### в) Перенос в приложение
- → App: **Критерий выбора якорной задачи расширен:** «срочно / важно / в радость» — три равноправных фильтра-подсказки; иногда приложение само предлагает: «Может, сегодня якорная — что-то приятное?» (особенно после серии тяжёлых дней).
- → App: **Reflect в закрытии дня:** один вопрос, одна кнопка: «Что-то поменять на завтра?» [лимит меньше / напоминания тише / всё ок] — микронастройка системы без экрана настроек.
- → App: **Фичи как включаемые модули:** стрик-подобные визуализации, утренний выбор, статистика — всё opt-in/opt-out; онбординг включает минимум.

## 14. The Bullet Journal Method — Райдер Кэрролл (2018)

### а) Механики
1. **Rapid logging:** записи одной строкой с сигификаторами: • задача, ○ событие, – заметка; x — сделано, > — перенесено, < — в план будущего, зачёркнуто — неактуально.
2. **Migration (перенос) как фильтр:** в конце месяца незакрытые задачи переписываются вручную в новый список. Ключ: цена ручного переписывания заставляет спросить «оно вообще того стоит?» — не стоит → вычеркнуть без вины. Перенос — это осознанное решение, а не признак провала.
3. **Collections:** тематические списки под любую нужду; структура создаётся по мере необходимости, а не заранее.
4. **Future log:** парковка дел будущих месяцев без детализации.
5. **Daily log создаётся в день исполнения,** а не заранее на неделю — план не успевает протухнуть.
6. **Reflection утром и вечером** — встроена в сам акт ведения журнала.
7. Автор открыто говорит, что метод вырос из его СДВГ — это система, спроектированная под дефицит внимания изначально.

### б) Применимость к профилю
- ✅ Migration — самая важная механика для переноса: просроченная задача не «горит красным», а попадает в ритуал пересмотра, где перенос и удаление — равноправные достойные исходы. Это готовая философия «амнистии просрочки».
- ✅ Daily log в день исполнения — совпадает с «утренним выбором 3»; заранее расписанные дни — источник каскадных просрочек.
- ✅ Сигификатор «>» (перенесено) как нейтральный, безоценочный статус — прямо в UI: перенос ≠ провал.
- ⚠️ Бумажный журнал целиком — требует дисциплины ведения; сам Кэрролл признаёт, что пустые недели в журнале — норма. В приложении «пустой день» тоже должен быть нормой (нулевое состояние без упрёка).
- ❌ Противопоказаний мало; разве что коллекционирование красивых разворотов (культура вокруг BuJo) — прокрастинация обустройством системы. В приложении: минимум кастомизации на старте.

### в) Перенос в приложение
- → App: **Амнистия как ритуал миграции:** раз в неделю (или при накоплении N просроченных) — экран «Разбор хвостов»: каждая просроченная задача по одной, три кнопки — «всё ещё нужно → в новую неделю», «когда-нибудь», «уже неактуально → удалить». После разбора счётчик просрочки = 0. Никакого списка «просрочено 47» на главном экране — класс «просрочено» вообще не существует как публичное состояние.
- → App: **Статус «перенесено» нейтрален:** без красного цвета, без счётчика переносов на видном месте. (Внутренняя аналитика переносов — только для алгоритма подсказок: «эта задача переносится 4-ю неделю — может, в когда-нибудь или порезать?»)
- → App: **Быстрые заметки наравне с задачами** в quick capture (не всё — задача; мысль можно просто записать, сигификатор проставится при разборе).

## 15. The Now Habit — Нил Фьоре (1988)

### а) Механики
1. **Прокрастинация — не лень, а механизм защиты** от угрозы самооценке: «моя ценность = мой результат» → задача становится экзистенциальной угрозой → избегание снижает тревогу → закрепляется. Лечится снижением угрозы, а не усилением давления.
2. **Язык выбора:** замена «я должен» → «я выбираю», «я обязан закончить» → «когда я могу начать?». «Должен» активирует позицию жертвы и сопротивление.
3. **Unschedule (антирасписание):** сначала в календарь вносятся отдых, сон, еда, спорт, развлечения и обязательства — работа над проектом вписывается в оставшиеся дыры. Работа получает дефицитный слот → снижение давления «всё свободное время должно быть работой».
4. **Запись только начатого:** в Unschedule фиксируются только фактически отработанные блоки ≥30 мин, и только после того, как они случились. Награда за старт, не за завершение.
5. **Правило 30 минут:** обещание себе — всего 30 минут качественной работы, потом легальный отдых/переключение.
6. **Guilt-free play:** отдых планируется первым и не зарабатывается. Вина за отдых — топливо прокрастинации (отдых с виной не восстанавливает → ресурса нет → работа страшнее → круг).
7. **Worry inventory:** «что худшее случится, если...» прописывается до конца с планом Б — тревога конкретизируется и сдувается.
8. **Фокус на старте, никогда на финише:** вопрос всегда «когда я начну следующие 30 минут», никогда «когда я всё закончу».

### б) Применимость к профилю
- ✅ Вся книга применима почти без фильтра — это единственная классическая система, построенная от психологии избегания, а не от оптимизации. Модель «прокрастинация = защита от угрозы» точно описывает и депрессивное избегание, и СДВГ-паралич.
- ✅ Unschedule — противоядие от «всё свободное время = долг работе», особенно при подработках.
- ✅ Награда за старт — фундамент для «поковырять 5 минут», кнопок начала, отметки «начата».
- ✅ Guilt-free play — прямое проектное требование: приложение никогда не предлагает работу в слоты отдыха и не показывает задачи в защищённое время.
- ⚠️ 30 минут для старта — при тяжёлых днях много; масштабировать вниз до 5–10.
- ❌ Противопоказаний нет.

### в) Перенос в приложение
- → App: **Копирайтинг без «должен»:** во всех текстах приложения нет слов «должен», «обязан», «не забудь», «пора». Напоминания формулируются как выбор/вопрос: «Хочешь начать 10 минут X?» Кнопки: «начну сейчас / позже / не сегодня» — все три легальны и без цветовой иерархии «правильного» ответа.
- → App: **Отслеживание стартов как first-class метрика:** «начал задачу» — событие, равноценное «завершил» в статистике и празднованиях. Экран статистики показывает «стартов на этой неделе: 9», а не только процент завершения.
- → App: **Защищённые слоты отдыха:** пользователь размечает время «моё» (вечер пятницы, утро воскресенья) — приложение в эти часы не показывает ни задач, ни напоминаний (кроме класса «критичное»), а на попытку открыть список говорит «Сейчас твоё время. Задачи подождут».

## 16. The Procrastination Equation — Пирс Стил (2010)

### а) Механики
1. **Уравнение мотивации:** Motivation = (Expectancy × Value) / (Impulsiveness × Delay). Мотивация падает, когда: не верю, что получится (E↓), задача скучна/бессмысленна (V↓), я импульсивен (I↑ — у СДВГ хронически), дедлайн далеко (D↑).
2. Из уравнения — 4 рычага, каждый со своими техниками:
   - **Expectancy ↑:** спирали успеха (серии посильных побед), рамка «уже получалось».
   - **Value ↑:** связать со смыслом, добавить игру/челлендж, temptation bundling (совмещать скучное с приятным), энергоменеджмент (скучное — на пик энергии).
   - **Impulsiveness ↓:** precommitment (заранее отрезать пути к отвлечению), автоматизация решений.
   - **Delay ↓:** дробить дальний дедлайн на ближние микродедлайны; делать цели конкретными и близкими (proximal subgoals).
3. **Гиперболическое дисконтирование:** ценность награды резко падает с отсрочкой; у импульсивных — круче. Дальние цели эмоционально «не существуют».
4. Прокрастинация коррелирует с импульсивностью сильнее, чем с перфекционизмом (по мета-анализам Стила) — т.е. это про регуляцию импульса, не про «высокие стандарты».

### б) Применимость к профилю
- ✅ Уравнение — готовый диагностический алгоритм: для залипшей задачи можно определить, какой множитель сломан, и применить соответствующий рычаг. Это структурная основа для «диагностики залипших задач» (дополняет P→A→M Фогга).
- ✅ Proximal subgoals — научное обоснование декомпозиции и «сегодняшнего» горизонта.
- ✅ Temptation bundling — легальный чит: скучное + подкаст/сериал.
- ⚠️ Спирали успеха — работают, но требуют, чтобы первые задачи были гарантированно посильными → связка с мини-версиями.
- ❌ Геймификация с очками/уровнями (Стил допускает) — для профиля осторожно: внешняя игровая мотивация выгорает, а сломанный «уровень» = ещё один заброшенный трекер. Только лёгкие формы без накопительных потерь.

### в) Перенос в приложение
- → App: **Диагностика залипшей задачи по 4 рычагам** (задача переносится ≥3 раз → предложить мастер): «Не верится, что получится? [→ порезать до смешного] Скучно? [→ с чем приятным совместить?] Отвлекаешься? [→ фокус-режим] Дедлайн далеко? [→ поставим микрошаг на завтра]».
- → App: **Поле «приманка»** у скучных повторяющихся задач: «делать под подкаст/музыку/сериал» — показывается в напоминании («Глажка + твой подкаст?»).
- → App: **Автомикродедлайны:** у задачи с дальним сроком приложение предлагает 2–3 промежуточные вехи; напоминания привязываются к вехам, не к финальной дате.

## 17. Four Thousand Weeks — Оливер Бёркеман (2021)

### а) Механики
1. **Жизнь ≈ 4000 недель; «успеть всё» математически невозможно.** Продуктивность-как-контроль — ловушка: чем эффективнее разгребаешь, тем больше наливается (email-эффект). Отправная точка — принять конечность, а не победить её.
2. **Fixed volume approach:** жёстко ограниченные ёмкости (например, максимум 3 активных проекта; закрытый список дня фиксированной длины). Новое — только когда что-то завершено/выброшено.
3. **Pay yourself first:** своё главное — первым куском времени, до чужих запросов.
4. **Strategic underachievement:** заранее выбрать сферы, в которых осознанно фейлишь (в этом квартале дом — на минималках), — вместо стихийного провала везде со стыдом.
5. **«Done list»:** вечером — список сделанного (вместо созерцания несделанного); при депрессии/тревоге — терапевтичен.
6. **Атерентность к процессу:** «radical incrementalism» — маленькими порциями, останавливаться, пока хочется продолжать; терпимость к «сезонам» продуктивности.
7. **Против «когда-нибудь разберусь со всем»:** clearing-the-decks (сначала разгребу мелочь, потом займусь главным) — самообман, мелочь бесконечна.

### б) Применимость к профилю
- ✅ Философский фундамент всего приложения: список никогда не будет пуст, и это не провал, а свойство реальности. Снимает вину системно, а не косметически.
- ✅ Fixed volume — то же, что лимиты по типам дней, но с обоснованием «это не костыль для слабых, это адекватность».
- ✅ Strategic underachievement — готовая фича: явный выбор «спящих сфер» превращает провал в решение. Решение не стыдно, провал стыдно — вся разница в наличии выбора.
- ✅ Done list — обязательный элемент закрытия дня для депрессивного профиля (когнитивное искажение депрессии — обесценивание сделанного; вечерний done list — прямая КПТ-техника против него).
- ❌ Противопоказаний нет; риск лишь в том, что философию «всё не успеешь» депрессия может прочитать как «всё бессмысленно» — тексты в приложении должны подавать это как освобождение, не как капитуляцию.

### в) Перенос в приложение
- → App: **Done list в закрытии дня:** экран «Сегодня было» — автособранный список выполненного (включая мини-версии, старты и категорию «восстановление»); при пустом списке — без укора: «День на выживание — тоже день. Завтра новый».
- → App: **Спящие сферы:** пользователь может перевести сферу (быт/блог/спорт...) в режим «на минималках до [дата]»: её задачи скрываются из планирования, привычки — на паузе без сброса статистики, в интерфейсе сфера помечена нейтрально («отдыхает»).
- → App: **Закрытый список дня:** после утреннего выбора список дня «запечатывается»; входящие в течение дня падают в inbox/завтра, а не в сегодня (кроме явного «нет, это сегодня» от пользователя).

## 18. How to Keep House While Drowning — Кейси Дэвис (2020)

### а) Механики
1. **Забота, а не мораль:** уборка/быт/задачи — морально нейтральны. Грязная посуда не делает тебя плохим; выполненная задача не делает хорошим. Единственный вопрос: «что сейчас позаботится обо мне?»
2. **Функционально, а не красиво:** цель пространства/системы — работать для тебя, а не выглядеть правильно. «Достаточно хорошо» — легитимный финал.
3. **5 вещей (five things tidying):** любой завал состоит из 5 категорий (мусор, посуда, бельё, вещи с местом, вещи без места) — разбирать по одной категории, а не «убрать комнату».
4. **Ритм, а не рутина:** не «каждый день в 8», а «примерно после ужина, когда получается»; пропуск не ломает ритм, потому что ритм определён нечётко by design.
5. **Closing duties:** короткий вечерний набор действий «для завтрашнего себя» (5–10 мин); формулировка — подарок себе-будущему, не обязанность.
6. **Правило 9 из 10:** если делаешь задачу в 9 случаях из 10 — ты «человек, который это делает»; 10-й раз не отменяет идентичность.
7. **Rest ethic:** отдых не зарабатывается; «ты не обязан заслуживать право сесть».
8. **Разрешение на костыли:** бумажные тарелки в тяжёлую неделю, сушка вместо глажки — снижение стандарта как акт заботы, а не деградация.

### б) Применимость к профилю
- ✅ Применимо всё; книга написана терапевтом с СДВГ ровно для состояния «depression + drowning». Это книга-камертон для tone of voice всего приложения.
- ✅ «Забота, а не мораль» — генеральный принцип копирайтинга и визуального языка: никакие состояния задач не окрашиваются морально (нет красного «просрочено», нет «you failed»).
- ✅ Правило 9 из 10 — числовое выражение отказа от стриков: идентичность держится на частоте, а не на непрерывности.
- ✅ Ритм vs рутина — обоснование «мягких» повторов: привычка «3 раза в неделю, когда получится» вместо «пн-ср-пт 8:00».
- ❌ Противопоказаний нет.

### в) Перенос в приложение
- → App: **Гибкие повторы как первый класс:** тип повтора «N раз в неделю» (без привязки к дням) — дефолт для привычек; жёсткие «каждый день в 8:00» — продвинутая опция с предупреждением.
- → App: **Метрика частоты вместо непрерывности:** карточка привычки показывает «8 из последних 10 раз» (скользящее окно), а не «стрик 0 🔥💀». Пропуски внутри окна визуально не выделяются.
- → App: **Closing duties как шаблон:** готовый вечерний чеклист из 3–5 мини-действий с формулировкой «подарок завтрашнему себе»; выполнение любого одного пункта — уже зачёт ритуала.
- → App: **Режим «выживание»:** отдельный тип дня ниже «плохого»: показывается только категория «восстановление» + 1 критичная задача максимум; всё остальное скрыто; включается одной кнопкой без объяснений.

## 19. Driven to Distraction (1994) / ADHD 2.0 (2021) — Эдвард Хэлловэлл, Джон Рейти

### а) Механики
1. **СДВГ — это дефицит регуляции внимания и мотивации, а не дефицит внимания:** интерес, новизна, вызов, срочность (interest-based nervous system) включают фокус там, где важность и долг — нет. Систему надо строить на интересе/новизне/срочности, а не на «важно».
2. **Внешняя структура — протез исполнительной функции:** списки, таймеры, напоминания, визуальные якоря, body doubling (работать рядом с кем-то) — не признак слабости, а очки для мозга.
3. **Time blindness:** будущее эмоционально не существует; работают только «сейчас» и «не сейчас» → любые дальние сроки обязаны конвертироваться в ближние сигналы.
4. **Пилот/штурман:** внешний человек (или сервис), помогающий планировать и сверяться — легитимная часть системы.
5. **Упражнения/движение** — самое доступное «лекарство» без рецепта: краткосрочно повышает дофамин/норадреналин, улучшает фокус на 1–3 часа.
6. **Стимуляция средой:** фоновая музыка, фиджеты, работа в кафе — дозируемый шум помогает (оптимум стимуляции у СДВГ выше).
7. **Гиперфокус — ресурс и риск:** умение входить в него на нужном и выходить с внешним будильником.
8. **«Никогда не волнуйся в одиночку»** и связь как терапия — социальная поддержка снижает тревогу и стыд, хронически сопровождающие СДВГ.
9. **VAST/«Феррари-мозг с велосипедными тормозами»:** рефрейминг диагноза без стыда; сильные стороны (креативность, энергия в интересном) — опора, а не утешительный приз.

### б) Применимость к профилю
- ✅ Interest-based nervous system — ключ к сортировке: подсовывать «новое/интересное/срочное» в моменты залипания, а не давить «важным».
- ✅ Внешняя структура как протез — оправдание максимальной автоматизации: приложение должно решать за пользователя всё, что можно решить без него (перенос хвостов, сортировка, буферы) — каждое сэкономленное решение = сэкономленный дефицитный ресурс.
- ✅ Time blindness → все дальние дедлайны переводятся в «сегодня/не сегодня» интерфейсно.
- ✅ Движение как лекарство → категория «восстановление» с приоритетом движения в плохие дни («10 приседаний разбудят голову — потом решишь про остальное»).
- ⚠️ Body doubling — в соло-приложении реализуем как виртуальный вариант (фокус-таймер «вместе», стрим-режимы) — опционально, во второй очереди.
- ❌ Противопоказаний нет; книги описывают целевую нейрофизиологию напрямую.

### в) Перенос в приложение
- → App: **Тег «интересное» и режим «залип»:** кнопка «не могу начать ничего» → приложение предлагает не топ-приоритет, а самое короткое/новое/интересное из списка (алгоритм: сортировка по новизне добавления + краткости + тегу «в радость»).
- → App: **Конвертация дальних сроков:** дедлайн > 3 дней никогда не показывается как дата; показывается ближайший микрошаг («сегодня: открыть документ»). Полная временная шкала — по запросу.
- → App: **Будильник выхода из гиперфокуса:** в фокус-таймере опция «жёсткий стоп» (нельзя продлить в один тап — требуется подтверждение с показом того, что дальше по плану: «через 15 мин — забрать вещи из химчистки»).
- → App: **Внешние сигналы времени:** напоминания «осталось 30 мин до выхода» для задач с жёсткой привязкой (класс «критичное»); таймеры видимого обратного отсчёта вместо абстрактного времени.

## 20. How to Change — Кэти Милкман (2021)

### а) Механики
1. **Fresh start effect:** временные вехи (понедельник, 1-е число, день рождения, возврат из отпуска) психологически «обнуляют» прошлые неудачи и повышают готовность начать — эффект измерен на больших данных.
2. **Temptation bundling** (её исследование): жёсткая связка соблазна с полезным (аудиокнига-триллер ТОЛЬКО в спортзале) — рост посещаемости до +50% в экспериментах.
3. **Commitment devices:** внешние ограничители — от мягких (обещание) до жёстких (штрафы, блокировки). Работают, но жёсткие несут риск: сорвался → штраф → стыд → выход из игры.
4. **Планирование с конкретикой (implementation intentions):** «что-где-когда» повышает исполнение; напоминание с привязкой к конкретному моменту эффективнее общего.
5. **Эластичность цели (emergency reserves):** цель «спортзал 5 раз в неделю» с 2 легальными джокерами пропуска устойчивее, чем «3 раза без права пропуска» — жёсткая цель при первом же срыве рушится целиком (исследование Милкман: цели с запасом прочности дают лучший долгосрочный результат).
6. **Giving advice:** дающий советы другим сам лучше следует цели (эффект «сам себе наставник»).
7. **Copy-paste prompts:** подсматривать и копировать конкретные приёмы успешных пиров.

### б) Применимость к профилю
- ✅ Fresh start — научное обоснование «амнистии»: обнуление хвостов к понедельнику/1-му числу — это не заметание под ковёр, а использование задокументированного психологического окна.
- ✅ Emergency reserves / джокеры — механика, разрешающая пропуск ДО того, как он случился: пропуск по джокеру — не срыв, а трата легального ресурса. Прямо снимает цикл «пропустил → вина».
- ✅ Implementation intentions — конкретика «что-где-когда» в напоминаниях.
- ⚠️ Commitment devices — только мягкие формы; денежные штрафы и публичные ставки для профиля токсичны (см. Антирекомендации).
- ❌ Социальное сравнение/копирование пиров — для депрессивного профиля сравнение с другими — топливо самокритики. Не встраивать лидерборды и «у других получается».

### в) Перенос в приложение
- → App: **Джокеры пропуска:** у каждой привычки — 1–2 джокера в неделю (визуально — «жетоны»); пропуск при наличии джокера списывает жетон и НЕ считается пропуском ни в какой статистике. Жетоны сгорают в конце недели (не копятся в долг).
- → App: **Fresh start-пуши (класс «опциональное»):** в понедельник утром / 1-го числа — «Чистая страница. Прошлая неделя закрыта. Выбрать 1–3 вещи на эту?» Особенно после плохой недели — но без упоминания, что она была плохой.
- → App: **Амнистия приурочена к вехам:** автопредложение разбора хвостов в конце недели/месяца использует fresh start окно.

# ЧАСТЬ II. СИНТЕЗ

Принципы отсортированы по числу систем, в которых механика встречается (номера систем — из Части I). **Порог доверия: 5+ систем = проверено временем, внедрять как ядро.**

## Принципы, проверенные временем (5+ систем)

**П1. Радикальное уменьшение шага / минимальная версия — 10 систем** (3, 4, 5, 6, 10, 11, 15, 16, 17, 18)
Barrier to entry снижается до уровня, при котором старт гарантирован даже без ресурса. Реализация: обязательное поле мини-версии, «поковырять 5 минут», stupid small, 30-секундные привычки, salami slicing, proximal subgoals, radical incrementalism.
*Ядро приложения — уже в концепции.*

**П2. Жёсткий лимит одновременного — 9 систем** (2, 4, 5, 7, 8, 9, 10, 11, 17)
1–3 главных на день/неделю, закрытые списки, fixed volume, «не более 3 привычек». Конвергенция на числе **3 для дня** (Бэйли, Трейси, частично Кови) и **1 для якоря** (Келлер, Кнапп).
*→ App: лимиты по типам дней: выживание 1, плохой 1–2, обычный 3, сильный 5; якорная — всегда одна.*

**П3. Снижение числа решений в моменте / внешняя структура — 9 систем** (1, 3, 6, 7, 11, 12, 14, 19, 20)
Решения принимаются заранее (вечернее планирование, implementation intentions, ритуалы входа, автоматизация) либо отдаются системе (автоперенос, автосортировка). Каждое решение в моменте — расход дефицитного ресурса.
*→ App: генеральное правило дизайна — «по умолчанию приложение решает само, пользователь может переопределить». Никогда наоборот.*

**П4. Пропуск — норма; важна скорость возврата, а не непрерывность — 8 систем** (3, 4, 8, 13, 14, 17, 18, 20)
Never miss twice, «возврат внимания — норма», правило 9 из 10, джокеры, ритм vs рутина, «hot mess is OK», нейтральный статус переноса.
*→ App: метрика возврата как главная метрика привычек; празднование возврата сильнее празднования регулярности.*

**П5. Немедленная награда / празднование малого — 7 систем** (3, 4, 6, 12, 15, 16, 17)
Celebration, small wins, награда за старт, done list, отметка выполнения как reward. При СДВГ отложенная награда не работает физиологически; при депрессии внутренний reward ослаблен — награду даёт интерфейс.
*→ App: хаптика/анимация при каждой отметке; done list вечером; «старт» — награждаемое событие.*

**П6. Якорение к событию/контексту, а не к астрономическому времени — 7 систем** (1, 3, 4, 6, 14, 19, 20)
Habit stacking, «после [якоря]», контексты GTD, триггеры Дахигга, «что-где-когда», компенсация time blindness.
*→ App: поле «после чего» у привычек; напоминания по событию-прокси, а не только по часам.*

**П7. Отдых/восстановление — часть системы, не награда — 7 систем** (2, 7, 8, 13, 15, 17, 18)
P/PC, shutdown ritual, scatterfocus, energize, guilt-free play, rest ethic, done list вместо руминации.
*→ App: категория «восстановление» первого класса; защищённые слоты; закрытие дня.*

**П8. Конкретизация следующего действия — 6 систем** (1, 6, 9, 11, 16, 20)
«Следующее физическое действие», implementation intentions, микрошаги, «что-где-когда». Аморфная формулировка = главный предиктор залипания.
*→ App: линтер формулировок; дальний дедлайн всегда показан как ближний микрошаг.*

**П9. Планирование по энергии, а не только по важности — 6 систем** (1, 7, 8, 11, 16, 19)
Фильтр энергии GTD, Biological Prime Time, тяжёлое на пик, interest-based выбор при залипании.
*→ App: поле «энергия» у задач + тип дня = матчинг; режим «залип» предлагает лёгкое/интересное.*

**П10. Регулярный пересмотр с легальным выбрасыванием — 6 систем** (1, 10, 14, 16, 17, 20)
Weekly review, migration, амнистия, «неактуально» как достойный исход, fresh start, устранение «неплохого».
*→ App: недельный ритуал разбора хвостов ≤5 мин; автопротухание Someday; счётчик просрочки не существует как публичное состояние.*

**П11. Дизайн среды > сила воли — 5 систем** (3, 4, 7, 13, 16)
Убрать трение для нужного, добавить для ненужного; precommitment; ритуалы входа; laser-тактики.
*→ App: поле «что подготовить»; фокус-режим; блокировка отвлечений в защищённый слот (мягкая, opt-in).*

**П12. Внешняя выгрузка головы (capture) — 5 систем** (1, 7, 12, 14, 19)
Inbox, лист отвлечений, rapid logging, парковка мыслей, протез рабочей памяти.
*→ App: quick capture ≤2 тапа отовсюду, включая фокус-режим.*

## Сильные принципы (3–4 системы)

- **П13. Награда за старт, а не за финиш** (11, 12, 15 + косвенно 5) → метрика стартов, «начата» как статус.
- **П14. Планирование с вечера / закрытие дня** (7, 11, 13, 14) → вечерний ритуал: done list + перенос хвостов + выбор до 3 на завтра.
- **П15. Недельный горизонт как основной** (2, 14, 17 + косвенно 20) → неделя первична, день — производный.
- **П16. Time blindness требует конвертации дальнего в ближнее** (16, 19, 20) → микродедлайны, «сегодня/не сегодня».
- **П17. Тон без морали и вины** (13, 15, 18 + косвенно 4) → копирайтинг «забота, не мораль»; ошибки — ошибки дизайна.
- **П18. Явный trade-off при перегрузе** (2, 10, 17) → zero-sum день, спящие сферы.
- **П19. Таймер как внешняя структура времени** (7, 12, 19) → фокус-таймер с гибкой длиной и будильником выхода.
- **П20. Буфер к оценкам времени** (10, 12 + косвенно 16) → автокоэффициент ×1.5 → персональный из статистики.

## Противоречия между системами и выбор для профиля

**Противоречие 1. Стрики (Клир п.6, культура BuJo-спредов) vs «важна частота и возврат» (Клир п.7, Дэвис, Милкман, Бэйли, Кнапп).**
Выбор: **против стриков, безоговорочно.** Для профиля с циклом «пропустил → вина → бросил» стрик — механизм катастрофизации одного пропуска. Замена: скользящее окно «N из 10» + метрика скорости возврата + джокеры. Сам Клир в главе про never miss twice фактически дезавуирует собственную стрик-механику.

**Противоречие 2. «Лягушка/самое тяжёлое первым» (Трейси, Келлер, Кови) vs «разгон от лёгкого» (Гайз, Фогг, Фьоре, спирали успеха Стила).**
Выбор: **разгон от лёгкого — дефолт; лягушка — опция сильного дня.** При СДВГ+депрессии инициация — главное узкое место: старт с тяжёлого блокирует день целиком. Алгоритм сортировки утром: 1–2 микропобеды → якорная задача на пике разгона. В сильный день пользователь может включить «сначала главное».

**Противоречие 3. Полная система с обязательными ритуалами (GTD, time blocking Ньюпорта) vs меню независимых тактик (Кнапп, Дэвис).**
Выбор: **меню.** Монолитная система умирает целиком при первом сбое ритуала (классика заброшенных трекеров). Каждый модуль приложения (утренний выбор, закрытие дня, амнистия, таймер) работает независимо; пропуск любого ритуала ничего не ломает и не копит долг.

**Противоречие 4. Планировать по важности (Кови, Келлер, МакКеон) vs по интересу/энергии (Хэлловэлл, GTD п.7, Бэйли, Стил-Value).**
Выбор: **гибрид с приоритетом энергии.** Важность определяет ЧТО попадает в неделю (лимиты, якорная); энергия/интерес определяют ЧТО и КОГДА делается сегодня. Давить «важным» в момент низкого ресурса бесполезно (interest-based nervous system) — предлагать посильное.

**Противоречие 5. Жёсткое расписание (time blocking, Pomodoro-пуризм) vs гибкий ритм (Дэвис, BuJo, Unschedule).**
Выбор: **гибкий ритм + 1–2 жёстких якоря.** Жёсткое — только объективно жёсткое (встречи, дедлайны с внешними последствиями = класс «критичное»). Всё остальное — списки без времени. Полностью расписанный день у профиля ломается к полудню и хоронит остаток дня.

**Противоречие 6. Дальние цели как компас (Кови, Келлер: цепочка от 5 лет к сегодня) vs короткий горизонт (Бёркеман, Фьоре, Стил: дальнее эмоционально не существует).**
Выбор: **короткий горизонт в UI, дальние цели — опционально и спрятанно.** Максимум видимого горизонта — неделя. Цели/проекты существуют как контейнеры задач, но экраны «твоя миссия», «цель года» — не строить. При депрессии вопросы смысла жизни в планировщике вредны.

**Противоречие 7. «Сделай сразу за 2 минуты» (Аллен) vs «всё в один inbox, решения потом» (тот же Аллен, Бэйли-парковка).**
Выбор: **правило 2 минут только внутри сессии разбора inbox,** никогда — в течение дня (риск цепочки отвлечений при СДВГ).

# ЧАСТЬ III. АНТИРЕКОМЕНДАЦИИ

Механики из разобранных книг, которые для профиля «СДВГ + депрессия + история заброшенных трекеров» вредны. Не реализовывать, включая «по многочисленным просьбам».

**А1. Стрики / don't break the chain** (Atomic Habits п.6; культура Seinfeld method).
Один пропуск обесценивает недели работы → катастрофизация → «система сломана» → уход. Депрессия гарантирует нулевые дни; стрик превращает их в приговор. Замена: П4 (окно «N из 10», скорость возврата, джокеры).

**А2. Публичные обязательства, ставки, штрафы** (жёсткие commitment devices у Милкман; accountability partner в жёсткой форме у Трейси).
Для профиля стыд — не мотиватор, а причина выхода: сорвался публично → избегание не задачи, а самого приложения. Исследования самой Милкман показывают: жёсткие устройства эффективны у дисциплинированных и токсичны при срывах. Никаких «поделись прогрессом», лидербордов, штрафов.

**А3. Красная просрочка и счётчики долга** (стандарт индустрии таск-менеджеров; дух ABCDE Трейси «A-задачи горят»).
Красный бейдж «47 просрочено» — экран вины, главный триггер удаления приложения. Просрочка не существует как публичное состояние; хвосты уходят в ритуал амнистии (BuJo-migration, П10).

**А4. Стартовать день с самого тяжёлого** (Eat That Frog; 4-часовой блок Келлера; big rocks first Кови — как дефолт).
Блокирует инициацию у СДВГ+депрессии; несделанная утренняя лягушка отравляет весь день. Как opt-in для сильных дней — допустимо.

**А5. Автоусложнение и прогрессия нагрузки** (Goldilocks rule Клира в автоматике; «повышай планку» Трейси).
Алгоритм «ты справляешься — вот +20%» ломает хрупкое равновесие: планка калибруется по худшему дню (Гайз), повышение — только вручную пользователем.

**А6. Длинные обязательные ритуалы** (Weekly review GTD 1–2 часа; полный time blocking Ньюпорта).
Ритуал дороже 5 минут будет пропущен, пропуск монолитного ритуала валит систему. Все ритуалы ≤5 мин, все опциональны, ни один не копит долг.

**А7. Экраны миссии, видения, целей на годы** (Кови «начинай с конца»; Келлер goal setting to the Now).
При депрессивном эпизоде вопрос «какова цель твоей жизни» утяжеляет состояние и парализует. Горизонт UI — неделя (Противоречие 6).

**А8. Идентичность через результат** (identity-based habits Клира в негативном прочтении).
«Каждое действие — голос за то, кто ты» при депрессии инвертируется в «пропуск — доказательство, что я ничтожество». Идентичность в текстах приложения — только через правило 9 из 10 Дэвис (частота, не непрерывность) и только позитивно.

**А9. Аннулирование прерванного** (Pomodoro: «прерванный помидор не считается»).
Любая механика, обнуляющая частичный прогресс, воспроизводит цикл вины. Всё частичное фиксируется как прогресс: минуты фокуса, старты, мини-версии.

**А10. Социальное сравнение** (copy-paste prompts Милкман, соревновательная геймификация).
Сравнение с другими при депрессии — топливо самокритики («у всех получается, я один...»). Единственная допустимая точка сравнения — «я вчера vs я сегодня», и то мягко.

**А11. Накопительная геймификация с потерями** (очки/уровни/замерзающие стрики à la Duolingo).
Потеря накопленного = та же вина в игровой обёртке. Допустимы только ненакопительные награды момента: конфетти, тёплая фраза, растущее дерево, которое не умирает.

# ЧАСТЬ IV. ТРИ КНИГИ ДЛЯ ПРОЧТЕНИЯ ЦЕЛИКОМ

**1. «How to Keep House While Drowning» — Кейси Дэвис.**
Приоритет №1: единственная книга списка, написанная именно для состояния пользователя (функционирование при депрессии/СДВГ), а не для его улучшенной версии. Короткая (можно прочесть за 2 вечера даже без ресурса — сама структура книги учитывает это: автор разрешает читать главы вразброс и пропускать). Даёт главное — смену рамки «мораль → забота», без которой любые механики снова станут кнутом.

**2. «The Now Habit» — Нил Фьоре.**
Приоритет №2: объясняет механизм собственной прокрастинации (защита от угрозы самооценке) — понимание механизма снимает 50% самообвинений. Unschedule и guilt-free play — практики, работающие при полной занятости с подработками, когда кажется, что «всё время должно быть полезным».

**3. «ADHD 2.0» — Хэлловэлл, Рейти.**
Приоритет №3: короткая, современная (в отличие от 400-страничной Driven to Distraction), покрывает нейробиологию, рефрейминг без стыда и практики (движение, внешние структуры, interest-based подход). Помогает перестать мерить себя нейротипичной линейкой — что для пользователя с чертами СДВГ и опытом «со мной что-то не так» терапевтично само по себе.

*Почему не Atomic Habits / GTD: это книги «для сильных дней» — системы, требующие ресурса на собственное обслуживание. Их лучшие механики уже извлечены в этот документ; чтение целиком добавит в основном давление стандартов, которым профиль сейчас не соответствует.*
